# 栈的缺页行为澄清

## 关键问题

**"如果一直 push stack，不也是会一直产生 PF 吗？"**

答案：**是的！** 这是一个重要的澄清。

---

## 两种不同的场景

### 场景 1：重复访问相同栈深度（0 次后续缺页）

```c
void same_depth() {
    char buf[4096];  // 1 页（4KB）
    buf[0] = 'x';    // 访问
}

// 重复调用 1000 次
for (int i = 0; i < 1000; i++) {
    same_depth();
}
```

**缺页行为**：

```
第 1 次调用：
    rsp: 0x7FFFF000 → 0x7FFFE000
    访问 0x7FFFE000 → 缺页 #1 → 分配物理页
    页表：[0x7FFFE000 → 物理页 0x12345000]

第 2 次调用：
    rsp: 0x7FFFF000 → 0x7FFFE000（相同位置）
    访问 0x7FFFE000 → 页表已有映射 → 无缺页 ✓

第 3-1000 次：
    相同 → 无缺页 ✓

总缺页：1 次
```

### 场景 2：持续增长栈深度（每次都缺页）

```c
void grow_stack(int depth) {
    char buf[4096];  // 每层递归占 1 页
    buf[0] = 'x';
    if (depth > 0) grow_stack(depth - 1);
}

// 深度递归
grow_stack(1000);  // 1000 层
```

**缺页行为**：

```
第 1 层递归：
    rsp: 0x7FFFF000 → 0x7FFFE000
    访问 0x7FFFE000 → 缺页 #1

第 2 层递归：
    rsp: 0x7FFFE000 → 0x7FFFD000
    访问 0x7FFFD000 → 缺页 #2（新页！）

第 3 层递归：
    rsp: 0x7FFFD000 → 0x7FFFC000
    访问 0x7FFFC000 → 缺页 #3（新页！）

...

第 1000 层：
    访问新页 → 缺页 #1000

总缺页：1000 次
```

**关键**：每层递归访问**新的虚拟页**，所以每次都缺页！

---

## 栈的物理页缓存到底优化了什么？

### 优化的是：重复访问已访问过的区域

```c
// 第一次深度递归
grow_stack(1000);  // 触发 1000 次缺页
                    // 但页表映射保留

// 第二次相同深度递归
grow_stack(1000);  // 0 次缺页！
                    // 所有虚拟页已映射
```

**可视化**：

```
第一次 grow_stack(1000)：
    栈深度：0 → 1000 层
    页表状态：
        [0x7FFC0000 → 物理页 1]  ← 第 1000 层（最深）
        [0x7FFC1000 → 物理页 2]  ← 第 999 层
        ...
        [0x7FFFE000 → 物理页 1000]  ← 第 1 层
    缺页次数：1000 次

递归返回：
    栈深度：1000 → 0
    rsp 回到 0x7FFFF000
    但页表映射保留！
        [0x7FFC0000 → 物理页 1]  ← 仍在
        [0x7FFC1000 → 物理页 2]  ← 仍在
        ...

第二次 grow_stack(1000)：
    栈深度：0 → 1000 层
    访问 0x7FFFE000 → 页表已有 → 无缺页
    访问 0x7FFFD000 → 页表已有 → 无缺页
    ...
    访问 0x7FFC0000 → 页表已有 → 无缺页
    缺页次数：0 次
```

### 不优化的是：访问更深的栈区域

```c
// 第一次：深度 1000
grow_stack(1000);  // 1000 次缺页

// 第二次：深度 2000（更深！）
grow_stack(2000);  // 1000 次已有映射 + 1000 次新缺页
                    //                   ─────────────
                    //                   访问新页，必须缺页
```

## 与堆的本质区别

### 栈：页表映射持久

```c
for (int i = 0; i < 1000; i++) {
    char buf[4096];
    buf[0] = 'x';
}

缺页分析：
    第 1 次：访问 0x7FFFE000 → 缺页 #1 → 页表映射建立
    第 2 次：访问 0x7FFFE000 → 页表仍在 → 无缺页
    ...
    第 1000 次：访问 0x7FFFE000 → 页表仍在 → 无缺页

总缺页：1 次
原因：页表映射持久，不删除
```

### 堆：页表映射临时

```c
for (int i = 0; i < 1000; i++) {
    void *p = mmap(NULL, 4096, ...);  // 创建 VMA
    ((char*)p)[0] = 'x';               // 访问 → 缺页
    munmap(p, 4096);                   // 删除 VMA + 页表
}

缺页分析：
    第 1 次：
        mmap() → VMA 创建
        访问 → 缺页 #1 → 页表映射建立
        munmap() → 删除页表 ← 关键！

    第 2 次：
        mmap() → 新 VMA（可能不同地址）
        访问 → 缺页 #2 → 重新建立页表
        munmap() → 删除页表

    ...

总缺页：1000 次
原因：每次 munmap 都删除页表
```

## 真实性能对比

### 测试 1：相同栈深度重复调用

```c
// 栈
void stack_test() {
    for (int i = 0; i < 100000; i++) {
        char buf[1024];
        buf[0] = i;
    }
}

// 堆
void heap_test() {
    for (int i = 0; i < 100000; i++) {
        void *p = malloc(1024);
        ((char*)p)[0] = i;
        free(p);
    }
}
```

**预期结果**：
```
栈：
    缺页：1 次（首次访问该页）
    时间：~5 ms

堆（mmap 方式，假设 malloc 阈值低）：
    缺页：~100,000 次
    时间：~500 ms

差异：100 倍
```

### 测试 2：持续增长的栈深度

```c
// 持续深度递归
void deep_stack(int depth) {
    char buf[4096];
    buf[0] = 1;
    if (depth > 0) deep_stack(depth - 1);
}

// 第一次
deep_stack(10000);  // 10,000 次缺页

// 第二次（相同深度）
deep_stack(10000);  // 0 次缺页

// 第三次（更深）
deep_stack(20000);  // 10,000 次缺页（新区域）
```

**关键**：
- 访问**新页**：必然缺页（栈和堆都一样）
- 访问**已访问过的页**：栈无缺页，堆有缺页（页表已删除）

## 内核代码验证

### expand_downwards 不删除页表

```c
// mm/vma.c:3023
int expand_downwards(struct vm_area_struct *vma, unsigned long address)
{
    if (address < vma->vm_start) {
        vma->vm_start = address;  // 只修改 VMA 边界
        // 关键：不调用 unmap_page_range()
        // 已有的页表映射保留
    }
}
```

### munmap 删除页表

```c
// mm/mmap.c:1067
int do_munmap(...)
{
    unmap_page_range(vma, start, end, ...);  // 删除页表项
    free_pgtables(...);                       // 释放页表
    remove_vma(vma);                          // 删除 VMA
}
```

## 修正后的结论

### 1. 访问新页：栈和堆都会缺页

```
持续 push 新的栈页：
    每个新页首次访问 → 缺页

持续 malloc 新内存：
    每次 malloc → 缺页

在这个场景下，栈和堆的缺页次数可能相近
```

### 2. 重复访问已访问区域：栈快，堆慢

```
栈（相同深度重复调用）：
    第 1 次：缺页
    第 2-N 次：无缺页（页表保留）

堆（相同大小重复 malloc/free）：
    每次：缺页（页表每次删除）
```

### 3. 栈的优势是"历史访问的缓存"

**不是**：栈永远不缺页
**而是**：栈访问过的区域，页表映射会保留

**类比**：
- 栈：像浏览器缓存，访问过的页面保留在本地
- 堆：像无缓存模式，每次都重新下载

### 4. 实际应用中的差异

**典型栈使用**（重复调用相同深度函数）：
```c
for (int i = 0; i < 1000000; i++) {
    process_item();  // 相同栈深度
}
// 极少缺页
```

**典型堆使用**（频繁 malloc/free）：
```c
for (int i = 0; i < 1000000; i++) {
    void *p = malloc(1024);
    process(p);
    free(p);
}
// 大量缺页（如果用 mmap 方式）
```

## 总结

你的质疑完全正确！

| 场景 | 栈 | 堆 | 差异 |
|------|----|----|------|
| **持续访问新页** | 每次缺页 | 每次缺页 | **无差异** |
| **重复访问已访问页** | 首次缺页，后续无缺页 | 每次缺页 | **巨大差异** |

**关键纠正**：
- ❌ 栈不会缺页
- ✅ 栈访问新页时仍会缺页
- ✅ 栈的优势是**页表持久性**，重复访问已访问区域时无缺页
- ✅ 实际应用中，函数调用模式多是重复相同深度，因此栈表现出"快"

**持续 push 新页**：栈也会持续缺页，这是正确的观察！
