## 核心指令列表

### 1. 栈空间分配（只执行一次）

| 地址 | 指令 | 说明 |
|------|------|------|
| `e58` | `sub sp, sp, #0x30` | 分配 48 字节局部变量空间 |
| **`e5c`** | **`sub sp, sp, #0xc8, lsl #12`** | **⭐ 一次性分配 800KB = 0xc8000 字节** |

### 2. 外层循环入口（重复10次）

| 地址 | 指令 | 说明 |
|------|------|------|
| `ecc` | `mov w6, #0xa` | w6 = 10（外层循环计数器初始化）|
| **`ed0`** | **`add x0, sp, #0x28`** | **⭐ x0 = 数组基地址 = sp + 0x28** |
| `ed4` | `mov w1, #0x42` | w1 = 0x42（初始写入值）|

**关键点**：地址 `ed0` 是外层循环的入口，每次迭代都从这里开始，由于 `sp` 不变，所以 `x0` 每次都指向相同地址。

### 3. 内层循环体（遍历200页）

| 地址 | 指令 | 说明 |
|------|------|------|
| **`ee0`** | **`strb w1, [x0]`** | **写页首（触发缺页）** |
| **`ee4`** | **`strb w1, [x0, #4095]`** | **写页尾（确保整页被映射）** |
| `ee8` | `add w1, w1, #0x1` | 递增写入值 |
| **`eec`** | **`ldrb w3, [x0]`** | **读页首** |
| **`ef0`** | **`add x0, x0, #0x1, lsl #12`** | **⭐ 移动到下一页（+4096）** |
| `ef4-efc` | 累加到全局计数器 | 防止编译器优化 |
| **`f00`** | **`ldurb w3, [x0, #-1]`** | **读页尾** |
| `f04-f0c` | 累加到全局计数器 | 防止编译器优化 |
| **`f10`** | **`cmp x5, x0`** | **检查是否到达数组末尾** |
| **`f14`** | **`b.ne ee0`** | **⭐ 跳回内层循环开始** |

### 4. 外层循环控制

| 地址 | 指令 | 说明 |
|------|------|------|
| **`f18`** | **`subs w6, w6, #0x1`** | **⭐ 外层计数器递减** |
| **`f1c`** | **`b.ne ed0`** | **⭐⭐⭐ 跳回外层循环入口（绕过栈分配）** |

**最关键的指令**：`f1c` 跳转到 `ed0`，而不是函数开始，这意味着跳过了地址 `e5c` 的栈分配代码！

### 5. 栈空间恢复（只执行一次）

| 地址 | 指令 | 说明 |
|------|------|------|
| `fa8` | `add sp, sp, #0x30` | 恢复 48 字节局部变量空间 |
| **`fb0`** | **`add sp, sp, #0xc8, lsl #12`** | **⭐ 恢复 800KB 栈空间** |
| `fbc` | `ldp x29, x30, [sp], #32` | 恢复帧指针和返回地址 |
| `fc0` | `ret` | 返回 |

## 关键验证点

### ✅ 验证1：栈空间只分配一次
- **分配指令**：地址 `e5c` 的 `sub sp, sp, #0xc8, lsl #12`
- **执行次数**：1次（在外层循环之前）

### ✅ 验证2：外层循环不重新分配栈
- **外层循环入口**：地址 `ed0`
- **循环跳转**：地址 `f1c` 的 `b.ne ed0`
- **跳转目标**：直接跳到 `ed0`，跳过了 `e5c` 的栈分配代码

### ✅ 验证3：数组基地址固定不变
- **计算基地址**：地址 `ed0` 的 `add x0, sp, #0x28`
- **sp 的值**：在整个外层循环（10次迭代）中保持不变
- **结果**：每次迭代 `x0` 都是相同的地址，访问相同的 800KB 栈内存

### ✅ 验证4：内层循环触发缺页
- **写操作**：`ee0` (写页首) + `ee4` (写页尾)
- **缺页时机**：
  - 第1次外层循环：访问200个新页 → 触发约200次缺页
  - 第2-10次外层循环：访问相同的200页 → 页表已存在，缺页 ≈ 0
- **总缺页数**：约200次

## 性能分析

### 内存访问模式

```
第1次外层循环:
  sp + 0x28 + 0x0000 ← 第1页（缺页）
  sp + 0x28 + 0x1000 ← 第2页（缺页）
  ...
  sp + 0x28 + 0xc7000 ← 第200页（缺页）

第2次外层循环:
  sp + 0x28 + 0x0000 ← 第1页（命中，sp 未变！）
  sp + 0x28 + 0x1000 ← 第2页（命中）
  ...
  sp + 0x28 + 0xc7000 ← 第200页（命中）

...重复10次...
```

### 对比场景2

- **场景2**：顺序访问400个不同的页 → 400次缺页
- **场景3**：重复访问200个相同的页10次 → 200次缺页

## 总结

汇编代码完美验证了优化假设：

1. ✅ 栈空间在函数序言中一次性分配（`e5c`）
2. ✅ 外层循环跳转绕过栈分配代码（`f1c → ed0`）
3. ✅ sp 在整个外层循环中保持不变
4. ✅ 每次迭代访问相同的 800KB 栈内存
5. ✅ 实现了栈空间复用机制

这是编译器**循环不变量外提**（Loop-Invariant Code Motion）优化的典型案例。
